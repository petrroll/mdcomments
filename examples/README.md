# mdcomments вЂ” Cross-Parser Examples

Side-by-side comparison of how the [mdcomments](https://github.com/petrroll/mdcomments) syntax renders across five Markdown systems, both with default settings (graceful degradation) and with a purpose-built plugin that renders a full comment sidebar UI.

> **Note:** The plugins in this directory were generated by an LLM and have not been manually reviewed. They serve solely as illustrative examples of how mdcomments integration could work across different parsers.

## Outputs

After running `build-all.sh`, each system directory contains:

- `output-default.html` вЂ” Rendered with the system's default footnote support. Comment threads appear as numbered footnotes (graceful degradation).
- `output-plugin.html` вЂ” Rendered with the mdcomments plugin. Comment threads appear in a styled sidebar with author avatars, timestamps, and status badges.
- `output-default.pdf` / `output-plugin.pdf` вЂ” PDF versions (Pandoc and MkDocs only, requires WeasyPrint).

Open `dist/results.html` to view all outputs side-by-side in a tabbed interface.

## How It Works

### Default (No Plugin)

mdcomments syntax uses standard Markdown primitives вЂ” footnotes (`[^c-id]`), blockquotes (`>`), and @-mentions (`@author`). In any parser that supports footnotes, comments degrade gracefully to numbered footnotes at the bottom of the page. The comment content (authors, dates, replies) is still readable inside the footnote body.

### With Plugin

Each plugin:
1. **Detects** `c-` prefixed footnote definitions and parses their structured content (metadata, author entries, blockquoted replies)
2. **Replaces** inline footnote references with comment indicator badges (рџ’¬)
3. **Transforms** `==highlighted text==` with comment-aware `<mark>` elements
4. **Renders** comment threads as sidebar cards with author avatars, timestamps, status badges, and threaded replies
5. **Injects** the sidebar layout CSS for the comment UI
